# 热门档口功能 — 开发修改计划（修订版）

## 一、需求概述

将首页标签栏中"全部"改为"热门"，实现两种视图：

| 选中标签 | 展示内容 |
|----------|----------|
| **热门** | 全校档口中综合得分最高的 **3 个**，带第一名/第二名/第三名标签 |
| **具体食堂**（如"东一食堂"） | 该食堂 TOP 3 档口（带排名标签）+ **剩余档口**（无排名标签） |

---

## 二、当前架构分析

### 首页标签栏（`src/app/page.tsx:170-194`）

当前有 5 个按钮：
- **全部**（`selectedCafeteriaId === 'all'`）
- **东一食堂** / **东二食堂** / **北一食堂** / **北二食堂**

当前数据流：
- 全部/食堂 → `GET /api/stalls/ranked?cafeteriaId=xxx` → stalls-ranked.ts
- 返回所有档口，按 `calculateStallScore` 排序

### 评分算法（`stalls-ranked.ts:8-26`）

```typescript
function calculateStallScore(avgRating, totalReviews, totalViews) {
  const ratingScore = avgRating * 20;                     // 评分基数满分100
  const reviewWeight = 0.7 + log(totalReviews+1) * 0.3;  // 评价数权重
  const viewWeight = 0.85 + log(totalViews+1) * 0.15;     // 浏览量权重
  return ratingScore * reviewWeight * viewWeight;
}
```

## 三、修改方案

### 核心理念：零后端改动

现有 `GET /api/stalls/ranked` 端点已经返回按 score 排序的完整档口列表，前端直接复用即可。所有变化发生在前端。

### 3.1 标签栏：文字修改

**文件**：`src/app/page.tsx:179`

```
"全部"  →  "热门"
```

### 3.2 数据获取：无需改动

两个场景均复用现有 `GET /api/stalls/ranked`：

| 场景 | 请求 | 说明 |
|------|------|------|
| 选中"热门" | `GET /api/stalls/ranked` | 返回全校所有档口（前端取前 3） |
| 选中食堂 | `GET /api/stalls/ranked?cafeteriaId=xxx` | 返回该食堂所有档口（前端拆为 top3 + 剩余） |

> 为什么取全部而非只取 3 条？因为食堂场景需要"剩余档口"，必须查询全部。为保持一致，"热门"场景也复用同一端点，前端 `.slice(0, 3)` 即可。

### 3.3 "热门"视图渲染（`selectedCafeteriaId === 'all'`）

```

│  │ 🥇 第一名                    │ │
│  │   风味小厨  ⭐ 4.8  128条评价  │ │ ← 正常列表样式
│  ├─────────────────────────────┤ │
│  │ 🥈 第二名                    │ │
│  │   麻辣香锅  ⭐ 4.5  96条评价   │ │
│  ├─────────────────────────────┤ │
│  │ 🥉 第三名                    │ │
│  │   牛肉面馆  ⭐ 4.3  72条评价   │ │

└─────────────────────────────────┘
```

- 只显示 3 条
- 每条左侧显示排名标签（第一名/第二名/第三名），替代原有的评分变化徽章
- 排名标签样式：带背景色的圆角标签
  - 第一名：金色背景 `#D97706`
  - 第二名：银灰色背景 `#9CA3AF`
  - 第三名：铜色背景 `#B45309`

- 标题改为 `{食堂名称}`（沿用当前"校园食堂"或改为食堂名）
- 前 3 条：带排名标签（第一名/第二名/第三名）
- 第 4 条起：不显示排名标签，与当前列表样式一致
- 如果该档口总数 ≤ 3，只显示 TOP 3 分区，无"其他档口"分区

| 位置 | 改动 |

## 六、验收标准

| # | 验收项 | 验证方式 |
|---|--------|---------|
| 1 | 标签栏"全部"改为"热门" | 首页标签栏第一个按钮文字为"热门" |
| 2 | 选中"热门"只显示 3 个档口 | 页面列出 3 个档口，左侧带排名标签 |
| 3 | 排名标签文字正确 | 三条依次标注"第一名""第二名""第三名" |
| 4 | 选中具体食堂显示 TOP 3 + 剩余 | 如东一食堂有 6 个档口：前 3 带标签，后 3 无标签 |

| 7 | 加载状态 | 数据加载中显示骨架屏 |
| 8 | 排名顺序与评分一致 | #1 得分最高，#2 次之，#3 最低（按 score 排序） |
| 9 | 点击跳转 | 点击任意档口进入 `/stalls/[id]` 详情页无误 |
| 10 | 构建通过 | `bun run build` 无报错 |
| 11 | SQLite兼容 | `bun run dev:local` 下正常运行 |
