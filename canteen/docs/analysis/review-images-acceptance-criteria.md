## 验收用例

### TC-1：上传单张图片并成功显示

**前置条件：** 已登录学生账号，位于评价发布页面

**操作步骤：**
1. 选择一个档口/菜品
2. 填写评分和评价内容
3. 点击图片上传按钮，选择 1 张 JPG 图片（< 5MB）
4. 确认上传成功后预览缩略图显示
5. 提交评价
6. 跳转到档口/菜品详情页
7. 查找刚发布的评价

**预期结果：**
- 评价评论区显示该图片（80x80 缩略图）
- 图片正确显示，无破损图标
- 图片 URL 格式正确

---

### TC-2：上传多张图片（3 张）并全部显示

**前置条件：** 同 TC-1

**操作步骤：**
1. 一次选择 3 张不同格式的图片（JPG、PNG、WebP）
2. 确认预览区域显示 3 张缩略图
3. 提交评价
4. 查看评论区

**预期结果：**
- 3 张图片全部显示
- 图片区域水平滚动可查看全部
- 每张图片均为 80x80，object-cover 裁剪

---

### TC-3：不上传图片提交评价

**前置条件：** 同 TC-1

**操作步骤：**
1. 不选择任何图片
2. 填写评分和内容
3. 提交评价
4. 查看评论区

**预期结果：**
- 评价正常显示
- 评论区没有图片区域（不显示空的图片容器）

---

### TC-4：上传超出 5MB 的图片

**前置条件：** 同 TC-1

**操作步骤：**
1. 选择一张 > 5MB 的图片
2. 观察前端响应

**预期结果：**
- 上传失败提示
- 不触发后端 500 错误
- 原有已上传图片不受影响

---

### TC-5：上传不支持的格式

**前置条件：** 同 TC-1

**操作步骤：**
1. 选择一张 GIF 或 BMP 格式图片
2. 观察前端响应

**预期结果：**
- 文件选择对话框应限制为 JPG/PNG/WebP
- 若强行选择，上传被拒绝

---

### TC-6：SQLite 模式全流程验证

**前置条件：** `DB_PROVIDER=local`，SQLite 数据库已初始化

**操作步骤：**
1. 执行 `bun run db:init:local && bun run db:seed:local`
2. 执行 `bun run dev:local`
3. 重复 TC-1 至 TC-3

**预期结果：**
- 所有测试用例通过
- 查看 `data/local.db` 中 `reviews.images` 列存储格式为 `["/api/uploads/xxx.jpg"]`（JSON 字符串）
- 无 `TypeError: Unexpected binding type` 错误

---

### TC-7：多端数据一致性验证

**前置条件：** 使用 Neon（PostgreSQL）

**操作步骤：**
1. 在菜品详情页创建带图片的评价
2. 在档口详情页查看同一评价
3. 打开"我的评价"页面查看
4. 使用商户账号登录，在评价管理页面查看

**预期结果：**
- 4 个展示位置的图片均正常显示
- 图片 URL 一致

---

### TC-8：生产环境（Vercel）图片加载验证

**前置条件：** 已部署到 Vercel（已配置 Blob 存储或等效方案）

**操作步骤：**
1. 在生产环境重复 TC-1 至 TC-3
2. 多次刷新页面验证图片是否稳定加载
3. 在不同设备/浏览器上验证

**预期结果：**
- 图片稳定显示，不出现 404
- 图片加载速度快（CDN 缓存）

## 验收检查清单

| # | 检查项 | 通过标准 |

| 8 | 图片加载失败有回退 UI | 显示占位图标而非破损图片 |
